# 单集质量检查清单

交付前逐项核对。先跑命令做结构门，再做人审。

先分清当前交付物：

- 整集 / 场景块正文：用下面的“场景块剧本检查”
- 5 秒单镜头文件：用下面的“单镜头检查”
- 不要拿 `review` 的场景块规则去检查 `shot-001.md`

## 场景块剧本检查

### 命令可自动检查

- [ ] 有 `场景X:` 场景块
- [ ] 场景标题带合法时长范围
- [ ] 每个场景包含环境空镜、主体、动作、光影、镜头、画质、台词
- [ ] 总字符数不超过 3000
- [ ] 正文没有短到只剩提纲感
- [ ] 没有空行
- [ ] 没有 `---` 分隔符
- [ ] 风险词已巡检
- [ ] 台词行里没有动作或语速说明
- [ ] `台词:` 后存在有效对白行

### 人工必查

- [ ] 本集是剧本，不是小说段落
- [ ] 默认 4-5 个场景
- [ ] 独立地点切换不超过 3 次
- [ ] 开场前 20% 已出现异常信息、即时冲突、认知钩子或高吸引力动作
- [ ] 结尾有卡点
- [ ] 3 秒内进入冲突
- [ ] 至少 3 个爽点
- [ ] 至少 1 个反转或信息升级
- [ ] 至少回应 1 个旧悬念，同时再压出 1 个新钩子或升级现有钩子
- [ ] 冲突绑定了可感代价，不是输了也像没事发生
- [ ] 中段至少有 1 次实感回报：翻盘、揭示、关系推进、资源落袋，至少一种
- [ ] 每场至少完成“目标 -> 阻碍 -> 变化”三步
- [ ] 对白和动作足够密，不是每场只讲一个信息点
- [ ] 台词短、狠、有信息量
- [ ] 没有无效对话和水词

### 人物人工复核

- [ ] 首场景主角外貌完整
- [ ] 后续场景只写服装或造型变化
- [ ] 角色没有说出自己不该知道的秘密
- [ ] 人物行为符合既有设定
- [ ] 角色没有 OOC

## 单镜头检查

适用：

- `compose-shots` 产出的 prompt 或正文
- `shot-001.md` 这类严格 5 秒交付

建议命令：

- `python3 scripts/episode_pipeline.py check-shot <镜头文件路径>`
- `python3 scripts/episode_pipeline.py review-shot <镜头文件路径>`

核对：

- [ ] 标题是 `镜头X: 地点(0-5秒)`，不是 `场景X: 地点(0-45秒)`
- [ ] 只覆盖当前 1 个镜头，没有偷带下一个镜头的结果
- [ ] 秒数是严格连续 5 秒区间
- [ ] `(动作)` 里只有 1 个主要动作节拍
- [ ] 台词默认 0-1 句，没有把下一镜头的台词塞进来
- [ ] `(主体)/(环境)/(动作)/(光影)/(镜头)/(画质)/台词:` 结构完整
- [ ] 没有把单镜头写成长段分镜小说
- [ ] 这个镜头单独拿出来，观众也能看懂动作目标和结果
- [ ] 如无台词，口型和推进是否已经由动作承担
- [ ] 如果标题没有地点名，先回到 `scene.md` 的 `## 当前场摘要` 补地点，再继续写镜头

## 风险词巡检

命令会提示这些词，但要人工判断是否真在偷写小说：

- `心想`
- `内心`
- `觉得`
- `感觉`
- `紧张`
- `难过`
- `伤心`
- `愤怒`
- `仿佛`
- `似乎`
- `空气里`
- `氛围`
- `意识到`

如果出现，优先改成动作、表情、道具、造型、灯光或镜头。

## 字数

- [ ] 超标时先删空行、分隔符、重复人物描述
- [ ] 再精简镜头和光影形容词
