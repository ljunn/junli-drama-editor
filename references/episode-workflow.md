# 单集创作工作流

默认按 5 个阶段执行。

## 阶段 0：准备

- 先跑 `preflight`，确保不是“文件在，但内容还是模板”
- 读取 5 个核心配置文件
- 检查 `state/角色状态.md`
- 检查 `state/伏笔列表.md`
- 确认要写的是第几集、标题是什么、核心事件是什么

## 阶段 1：状态恢复

写新一集前至少回答 4 个问题：

1. 谁知道重生、婚约、马甲或隐藏身份？
2. 上一集最后一个卡点是什么？
3. 当前最活跃的 2-3 个伏笔是什么？
4. 这一集的目标是打脸、反转、结盟、误会升级，还是揭示秘密？

恢复摘要至少要带出：

- 最近 2-3 集摘要
- 当前角色知情状态
- 当前活跃伏笔
- 下一集目标

## 阶段 2：场景规划

先写场景卡，再写剧本。

`plan` 生成出来的场景卡不是摆设。`compose` 会把这份场景卡直接打进 Prompt Pack，所以这里就是本集结构骨架。

每集默认：

- 4-5 个场景
- 3 秒进冲突
- 独立地点切换不超过 3 次
- 至少 3 个爽点
- 结尾必须有卡点

推荐节奏：

1. 开场冲突
2. 信息扩大 / 第二重冲突
3. 打脸或反杀
4. 关系变化或秘密强化
5. 卡点收束

## 阶段 3：正式剧本

剧本只写可拍内容：

- 时间 / 地点
- 人物出场和造型
- 动作
- 表情
- 台词
- 光影
- 镜头

不能直接写：

- “她很紧张”
- “空气里弥漫着压抑”
- “他心想”
- “她突然意识到”

要改成：

- “她攥紧手包，指甲嵌进皮面”
- “走廊顶灯忽明忽暗，她半张脸沉进影子”

## 阶段 4：质量检查

最少检查这些项：

- 先跑 `check/review` 做结构巡检
- 是否仍是剧本，不是小说
- 首场景主角外貌是否完整
- 后续场景是否只保留服装变化
- 场景块是否完整
- 台词是否短句、有用、无水词
- 是否提前泄露角色不该知道的秘密
- 是否超过 3000 字符
- 是否保留了结尾卡点

注意：

- `check/review` 擅长抓结构、格式和风险词
- 爽点够不够、卡点狠不狠、人物有没有 OOC，仍要人工判断

## 阶段 5：状态更新

写完一集后先跑 `finish`，它会自动更新：

- `state/剧集历史.md`
- `task_log.md`
- `state/角色状态.md` 中的待确认回写提醒
- `state/伏笔列表.md` 中的待确认回写提醒

然后再人工把知情状态、造型变化、伏笔推进/回收细化进正式表格。如果状态不回写，下一集一定会乱。
