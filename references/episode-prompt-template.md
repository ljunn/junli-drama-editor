# 单集 Prompt 模板

默认不要直接用整集 `compose`。

默认应使用 `compose-scenes` 先生成单场 Prompt Pack，再用 `compose-shots` 继续拆成单镜头 Prompt Pack，不要硬写整集。

只有在你明确需要整集输出时，才手动使用：

```bash
python3 scripts/episode_pipeline.py compose <项目目录> --episode-num <集数> --allow-full-episode
```

如果手动拼 Prompt，至少保证下面这些上下文都在。

```text
你正在续写平台向微短剧。若出现冲突，以状态文件 > 场景计划 > 历史摘要 > 上一集剧本为准。

[粘贴 series-bible.md 全文]
[粘贴 character-design.md 全文]
[粘贴 visual-bible.md 全文]
[粘贴 narrative-style.md 全文]
[粘贴 task_log.md 全文或关键段落]

当前要写的剧集：第X集
本集标题：[标题]
本集大纲：[核心事件]
目标时长：[时长]

最近 2-3 集摘要：
[粘贴最近 2-3 集摘要]

上一集结束后的角色状态：
[粘贴 state/角色状态.md 的相关部分]

当前活跃伏笔：
[粘贴 state/伏笔列表.md 中活跃伏笔]

本集场景卡：
[粘贴 runtime/episode-XXXX/plan.md]

如果要继续拆成单镜头：
- 先保存单场结果到 `runtime/episode-XXXX/scene-YY/scene.md`
- `scene.md` 里只保留当前场摘要和镜头表，不要写完整长场景正文
- 再读取其中的 `## 5秒镜头单元表`，并确保每一行都是 `0-5秒 / 5-10秒 / 10-15秒` 这种连续 5 秒区间
- 每次只生成一个镜头，保存到 `runtime/episode-XXXX/scene-YY/shot-001.md`

如果有：
- [粘贴 docs/分集梗概.md 的本集小节]
- [粘贴上一集剧本尾段]

请生成剧本，要求：
1. 总字符数≤3000；优先压缩视觉修饰词，不要把剧情推进压没
2. 默认分 4-5 个场景
3. 开场 3 秒内就要出事，不要先寒暄和解释背景
4. 每个场景至少完成“目标 -> 阻碍 -> 变化”三步，不能只停在聊天和说明信息
5. 默认每个场景 3-6 句有效对白；若对白更少，必须用动作结果补足推进
6. 至少落地 1 个硬信息揭示、1 个正面对撞、1 个主动反制、1 个强后果卡点
7. 爽点必须表现为当场占优、拆穿、截胡、证据到手、身份压制、局势翻转之一，不能只是“知道了一个消息”
8. 每个场景格式如下：
   场景X: 地点(起始秒-结束秒)
   【环境空镜Xs】描述
   (主体)角色外貌(首场景写全，后续只写服装变化)
   (环境)...
   (动作)...
   (光影)...
   (镜头)...
   (画质)...
   台词:
   【停顿Xs】
   角色名:"对话"
9. 台词不含动作和语速说明
10. 本集至少 3 个爽点，且至少 1 次真实代价或翻车
11. 不要空行，不要 --- 分隔符
12. 结尾必须留卡点
13. 不能让角色越权说出不该知道的信息
```
