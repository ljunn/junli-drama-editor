# 最终响应模板

交付时不要散。先说任务类型，再说结果，再说剩余人工项。

## 1. 新写 / 续写

推荐结构：

1. 这次处理的是哪一集、哪种模式
2. 是否已按流程完成 `preflight / resume / plan / compose`
3. 这次正文或 Prompt Pack 的核心结果
4. 是否需要状态回写或人工确认

示例：

```text
本次处理的是第 8 集，模式是 AI 视频版。已完成状态恢复和场景卡准备，当前创作包已可直接续写。

这次重点保住了女主知情优势和结尾卡点。结构检查可通过，但卡点力度和角色一致性仍建议人工复核。
```

## 2. 定向返修

推荐结构：

1. 这次改的是哪一集、哪个 `focus`
2. 明确只动了哪些场景或哪些句子
3. 说明没有整集重写
4. 如果还有残留风险，点出来

示例：

```text
这次处理的是第 6 集，focus 是 `hook`。我只改了最后一个场景和收尾三句台词，没有重写整集。

卡点已经从“普通悬念”改成“证据即将公开”的强后果型卡点。结构上没问题，但如果你要更狠，还可以再加一层身份压制。
```

## 3. 质检

推荐结构：

1. 先给结论：通过 / 不通过 / 结构通过但仍需人审
2. 再列最关键的 2-4 个问题
3. 最后说哪些项是自动检查不到的

示例：

```text
结构检查基本通过，但还不建议直接交。

主要问题是结尾卡点偏软、女主有两句台词在解释剧情、男主知情边界还需要人工确认。格式和字数没问题，爽点密度与人物一致性仍需人工复核。
```

## 4. 恢复状态

推荐结构：

1. 现在写到哪一集
2. 最近 2-3 集发生了什么
3. 当前角色知情状态和活跃伏笔
4. 下一步建议先做什么

## 5. 投稿资料

推荐结构：

1. 先说缺哪些
2. 再说已整理出哪些
3. 明确当前是不是整套可交

## 不要这样收尾

- 不要只说“搞定了”
- 不要只贴大段内容，不解释改动边界
- 不要把结构检查说成质量全通过
- 不要省略“还需要人工确认什么”
