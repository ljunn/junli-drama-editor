# 定向返修策略

返修不是重写。先判问题类型，再动最小范围。

## 1. 对白弱

症状：

- 台词像解释剧情
- 句子太长
- 没有身份压制、关系推进或信息差

优先改法：

1. 先删废话和口水词
2. 把解释句改成短句对抗
3. 每句台词只保留一个功能：压制、揭示、试探、反击

不要：

- 为了“更狠”把所有人都写成一个语气
- 在台词里塞动作或镜头说明

## 2. 卡点软

症状：

- 结尾只是普通悬念
- 观众没有“下一集必须点开”的压力

优先改法：

1. 优先重写最后一场或最后 3 句台词
2. 卡点尽量落在“证据将公开 / 身份将暴露 / 关系将翻面 / 权力即将失控”
3. 最后一击要带明确后果，不要只留模糊情绪

不要：

- 用“她愣住了”“门突然开了”这种空悬念糊弄

## 3. 爽点少

症状：

- 冲突有了，但没有压制反转
- 主角一直吃瘪，没有有效回击

优先改法：

1. 先补“信息差反杀”
2. 再补“身份压制”或“证据压制”
3. 把爽点尽量前置，不要全堆到结尾

不要：

- 把吵架当爽点
- 用长篇解释代替反击动作

## 4. 节奏拖

症状：

- 进入冲突太慢
- 中段重复表达同一信息

优先改法：

1. 开场 3 秒内把冲突主体、关系和异常动作打出来
2. 合并重复场景
3. 同一信息只保留一次最狠的表达

不要：

- 为了补信息量，重复写角色已经知道的内容

## 5. 小说化

症状：

- 出现心理词、氛围词、抽象情绪词
- 一段话拍不出来

优先改法：

1. 把心理改成动作
2. 把情绪改成表情、道具、灯光、镜头
3. 把抽象描述拆成可拍细节

不要：

- 直接写“她很难过”“空气里弥漫着压抑”

## 6. 超 3000 字

优先裁剪顺序：

1. 空行、分隔符
2. 重复人物外貌
3. 重复镜头和光影形容词
4. 无效对白
5. 不推动主冲突的中间动作

不要：

- 先删关键信息点和卡点

## 7. 知情越权 / 人物失真

症状：

- 角色提前说出不该知道的秘密
- 人物动机突然跳变

优先改法：

1. 先对照 `state/角色状态.md`
2. 把越权信息改成试探、误判或怀疑
3. 必要时把信息揭露往后移一场

不要：

- 用一句“她突然猜到了”硬跳过去

## 返修顺序

多问题同时存在时，默认顺序：

`state -> pacing -> hook -> dialogue -> format`

先修状态和结构，再修台词和字面格式。否则容易越改越乱。
