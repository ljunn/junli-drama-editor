# 投稿资料检查清单

做投稿资料时，先查齐全度，再决定补哪些，不要直接从零重写。

## 一、基础 6 件套

- [ ] `docs/短剧概况.md`
- [ ] `docs/人物小传.md`
- [ ] `docs/故事大纲.md`
- [ ] `docs/分集梗概.md`
- [ ] 样集剧本
- [ ] 状态说明

## 二、每件的最低标准

### 1. 短剧概况

- [ ] 有剧名
- [ ] 有类型
- [ ] 有总集数
- [ ] 有单集时长
- [ ] 有目标受众
- [ ] 有 300 字内梗概
- [ ] 有核心看点 / 付费点

### 2. 人物小传

- [ ] 核心角色至少 2 人
- [ ] 每人有外观、性格、身份、核心技能或金手指
- [ ] 能看出角色关系和人物卖点

### 3. 故事大纲

- [ ] 有背景
- [ ] 有切入点
- [ ] 有主线推进
- [ ] 有高潮
- [ ] 有结尾与后续钩子

### 4. 分集梗概

- [ ] 至少覆盖样集到最新集
- [ ] 每集能看出起因、经过、结果/卡点
- [ ] 分集目标和 `linenew.md` 不冲突

### 5. 样集剧本

- [ ] 至少有 1 集能代表整体气质
- [ ] 最好有最新完成集
- [ ] 格式不是小说，而是可拍剧本

### 6. 状态说明

- [ ] 有角色知情状态
- [ ] 有活跃伏笔
- [ ] 有已完成集数
- [ ] 有后续主线方向

## 三、常见缺口

优先补的顺序：

1. 短剧概况
2. 人物小传
3. 故事大纲
4. 分集梗概
5. 样集剧本
6. 状态说明

## 四、不要怎么做

- 不要一缺资料就整剧重写
- 不要把正文硬塞进投稿资料里
- 不要让投稿资料和当前项目状态相互矛盾

## 五、一句话规则

投稿资料的目标不是“写得多”，而是“让平台一眼看懂项目卖点、人物关系和可持续追更点”。
